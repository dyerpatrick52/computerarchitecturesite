Tab 1: Boolean Algebra & K-Maps

This is the mathematics behind digital circuits. Mastering simplification is key to creating efficient hardware.

Key Concepts

  • Axioms & Theorems: Boolean algebra is built on a set of rules. [cite_start]Key axioms include the identity element ($A \cdot 1 = A$) and complement ($A + A' = 1$)[cite: 10351, 10352]. [cite_start]Important theorems include idempotency ($A \cdot A = A$) and the null element ($A+1=1$)[cite: 10356, 10355].
    • [cite_start]
  • DeMorgan's Theorem: A critical tool for simplification[cite: 547, 904].
      [cite_start]
    • $\overline{A \cdot B} = \overline{A} + \overline{B}$ [cite: 549, 911]
      • [cite_start]
    • $\overline{A + B} = \overline{A} \cdot \overline{B}$ [cite: 550, 918]
  • Canonical Forms: These are standard ways to write Boolean functions.
      [cite_start]
    • Sum-of-Products (SOP): The function is expressed as a sum of minterms[cite: 601, 10390]. [cite_start]Example: $F(A,B,C)=\Sigma m(1,3,5,6,7)$[cite: 10392].
      • [cite_start]
    • Product-of-Sums (POS): The function is expressed as a product of maxterms[cite: 10723]. [cite_start]Example: $F(A,B,C)=\Pi M(0,2,4)$[cite: 10725].

Karnaugh Map (K-Map) Simplification

[cite_start]

A K-map is a graphical representation of a truth table used for simplifying logic functions[cite: 644].

    [cite_start]
  • Prime Implicant (PI): An implicant that cannot be covered by a more reduced (simpler) implicant[cite: 800]. [cite_start]The minimal SOP form of a function is always a sum of prime implicants[cite: 10602].
    • [cite_start]
  • Essential Prime Implicant (EPI): A prime implicant that covers at least one minterm ('1') that no other prime implicant can cover[cite: 802]. [cite_start]All EPIs *must* be included in the minimal solution[cite: 802].

Tab 2: Combinational Circuits

Key Components

  • Adders:
      [cite_start]
    • Half Adder: Adds two bits, producing a Sum and a Carry output[cite: 1063]. [cite_start]$S = A \oplus B$, $C_{out} = A \cdot B$[cite: 1074, 1070].
      • [cite_start]
    • Full Adder: Adds three bits (A, B, and a Carry-in), producing a Sum and a Carry-out[cite: 1084, 13740, 13687]. [cite_start]A full adder can be built from two half adders and an OR gate[cite: 13756].
    • [cite_start]
  • Decoders: A decoder converts an n-bit binary input into one of up to $2^n$ unique outputs[cite: 9763]. [cite_start]For a given input code, only one output line is active[cite: 9779].
    • [cite_start]
  • Multiplexers (MUX): A MUX is a data selector[cite: 9802]. [cite_start]It uses n select lines to route one of the $2^n$ data inputs to the output[cite: 9812].

Sequential Circuit Design Procedure

[cite_start]

This is a core process for the course[cite: 4807].

    [cite_start]
  1. State Diagram: Create a state diagram (Mealy or Moore) from the problem description[cite: 4809].
    2. [cite_start]
  2. State Table: Convert the diagram into a state table[cite: 4810].
  3. State Assignment: Assign a unique binary code to each state. [cite_start]This creates the **transition table**[cite: 4812].
    4. [cite_start]
  4. Excitation Table: Choose a flip-flop type and use its excitation table to determine the required inputs for each flip-flop[cite: 4813].
    5. [cite_start]
  5. Excitation Equations: Use K-maps to derive the simplified Boolean expressions for each flip-flop input and the circuit output(s)[cite: 4814].
    6. [cite_start]
  6. Draw the Circuit: Implement the equations with logic gates and flip-flops[cite: 4815].

Tab 2: Signed Number Arithmetic

2's Complement Representation

    [cite_start]
  • This is the most common system for signed numbers[cite: 11517]. [cite_start]A negative number is found by taking the 1's complement (inverting all bits) and adding 1[cite: 11469, 1909].
    • [cite_start]
  • The range of an n-bit number is $[-2^{n-1}, 2^{n-1}-1]$[cite: 11540, 1989].

Example: 2's Complement Circle

Overflow Detection

An overflow occurs if the result is outside the representable range. [cite_start]This happens if the carry-in to the sign bit is different from the carry-out of the sign bit ($C_{n} \oplus C_{n-1} = 1$)[cite: 2192, 11578].
What to Focus On: Be an expert in 2's complement. Know how to represent numbers, perform subtraction via addition, and reliably detect overflow.

Tab 3: Floating-Point Numbers (IEEE 754)

Single-Precision (32-bit) Format

[cite_start]

A number is represented as $value=(-1)^{sign}(1+\sum_{i=1}^{23}b_{-i}2^{-i}) \times 2^{(e-127)}$[cite: 11685].

[cite_start]Source: CEG2136_Lecture_6.pdf [cite: 11678, 2252, 2256, 2258]
    [cite_start]
  • Sign bit (1 bit): 0 for positive, 1 for negative[cite: 11693, 2259].
  • Exponent (8 bits): Stored in a **biased format**. [cite_start]The actual exponent is found by subtracting a bias of 127[cite: 11697, 2262].
  • Mantissa (23 bits): For normalized numbers, there is an implicit leading '1.' (the "hidden bit") [cite_start]that is not stored[cite: 11698, 2265].
What to Focus On: Practice the conversion process in both directions: from a decimal number to its 32-bit hex representation, and from a 32-bit hex representation back to decimal.

Tab 3: Basic Computer Architecture

Key Registers & Components

[cite_start]Source: CEG2136_Lecture_9.pdf [cite: 14095, 14100, 14102, 14104, 14115, 14116, 14130, 14139, 14141]
    [cite_start]
  • PC (Program Counter): Holds the address of the *next* instruction to be fetched[cite: 13380, 14067, 14483].
    • [cite_start]
  • AR (Address Register): Holds the memory address for a read or write operation[cite: 13378, 14067, 14482].
    • [cite_start]
  • IR (Instruction Register): Holds the instruction code currently being executed[cite: 13381, 14067, 14480].
    • [cite_start]
  • AC (Accumulator): A general-purpose register for holding operands and results[cite: 14010, 14067, 14475].

Instruction Formats and Addressing Modes

    [cite_start]
  • Direct Mode (I=0): The address part of the instruction contains the address of the operand[cite: 13985, 14185, 14384].
    • [cite_start]
  • Indirect Mode (I=1): The address part contains the address of a pointer to the operand[cite: 13987, 14187, 14387].
What to Focus On: Memorize the names and functions of the key registers. Be able to trace the contents of these registers through the fetch-decode-execute cycle for a single instruction.
